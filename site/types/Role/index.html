Role

Описание типа

Роль.

Поля

Имя поля Необходимость Тип данных Комментарий
code True string Код роли.
description False string Описание роли.
title True string Название роли.
is_default True boolean Признак роли, назначаемой по умолчанию при создании нового сотрудника.
В аккаунте может существовать только одна роль с is_default = true.
is_admin True boolean Признак роли со следующими характеристиками:
Может быть только один сотрудник с такой ролью.
Сотрудник с такой ролью не может быть удален или заблокирован.
Сотруднику с такой ролью нельзя изменить роль.
Роль имеет доступ ко всем объектам системы вне зависимости от значения настроек managed_departments и managed_sites.
В аккаунте может существовать только одна роль с is_admin = true.
is_full_by_default True boolean Если true, то при включении роли для сотрудника, поля managed_departments и managed_sites автоматически заполняются всеми сайтами и всеми отделами. В дальнейшем при добавлении сайта или отдела, он автоматически попадает в соответствующее поле сотрудника.
Если false, то при включении роли для сотрудника, поля managed_departments и managed_sites автоматически сбрасываются в пустое значение. В дальнейшем при добавлении сайта или отдела он НЕ попадает в соответствующее поле сотрудника.
permissions True Object Объект, содержащий перечень методов, соответствующих уровней разрешений и списка полей сущностей доступных в рамках этого метода.
Возможные значения уровней разрешений (level):
none – доступ запрещен;
user – только объекты текущего пользователя;
managed – объекты, связанные с отделами и сайтами, указанными в полях managed_departments и managed_sites соответственно.
routed – объекты типа чат, звонок или лид, адресованные на сотрудника.
full – все объекты.
Список доступных полей (allowed_fields) может быть 2-х типов:
строка со значением «all» – доступны все поля.
Массив строк – названий полей сущности или параметров метода.
Пример фрагмента объекта permissions:
"Employees" : {
"list" : {
"level": "user",
"allowed_fields": "all"
},
"update" : {
"level": "user",
"allowed_ fields": [
"first_name",
"last_name",
"email",
"password",
"phone",
"photo"
]
}
}
